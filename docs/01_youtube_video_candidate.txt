# 🎯 Module 01: 目标视频定向寻址与元数据采集 (Target Video Sourcing & Metadata Acquisition)

> **🎯 模块业务目标：**
> 本模块是全链路数据的源头。基于 YouTube Data API v3，我们将按照预先制定的“双重控制变量”战略，在全网定向检索三大控制组的视频候选人（Candidates）。通过提取视频的流量元数据（播放、点赞、评论量），为下游的【深度评论抓取 (Module 02)】提供高价值、高相关的靶向目标。

### 🔌 Cell 1 & 2 - 依赖导入与 API 引擎点火 (Environment & API Initialization)
* **职责**：
  1. 通过 `pyprojroot` 确立工业级绝对路径，保障工程在任何环境下的目录鲁棒性。
  2. 从 `.env` 文件安全加载 `YOUTUBE_API_KEY`。
  3. 实例化 Google API Client，打通本地与 YouTube 服务器的通信管道。

### 📜 Cell 3 - 战略检索配置装载 (Strategic Configuration Loading)
* **职责**：读取外部 `config/youtube_search.yaml` 配置文件。
* **业务意义**：将代码与业务逻辑解耦。在此处正式确立三大对比阵营（控制变量组）：
  * 🔴 `shein_labor_exploitation` (靶心组：SHEIN 道德指控)
  * 🔵 `non_shein_labor_exploitation` (对照组 A：竞品道德指控)
  * 🟢 `fast_fashion_environment` (对照组 B/基准线：行业宏观底噪)

### 🕷️ Cell 4 & 5 - 核心寻址引擎定义 (Core Sourcing Engine Definition)
* **职责**：构建两个极其关键的原子函数：
  1. `search_videos`: 负责分页 (Pagination) 与精确制导（限制语言为 `en`，类型为 `video`，过滤无效短片）。
  2. `get_video_stats`: 负责提取**视频级流量指标 (Video-level Engagement Metrics)**，包括：
     * 👁️ `viewCount` (播放量 - 衡量曝光基数)
     * 👍 `likeCount` (点赞量 - 衡量视频情绪赞同度)
     * 💬 `commentCount` (评论量 - 衡量舆论厚度，直接决定下游模块的数据量)
     * ⏱️ `duration` (时长解析 - 剔除过短或过长的无效内容)

### 🚀 Cell 6 - 自动化检索流水线 (Automated Sourcing Pipeline)
* **职责**：遍历 YAML 中定义的阵营列表，自动化执行 `[关键词检索] -> [视频 ID 获取] -> [元数据批量提取]` 的标准作业程序 (SOP)。
* **输出**：将 API 返回的松散 JSON 数据结构化为强大的 Pandas DataFrame，为每个控制组生成专属的数据资产。

### 💾 Cell 7 & 8 - 候选资产持久化与质量体检 (Asset Persistence & Quality Check)
* **职责**：
  1. 动态生成时间戳文件夹（如 `data_01_video_candidates_YYYYMMDD_HHMMSS`），实现数据版本的防覆写隔离。
  2. 将各阵营的视频元数据固化为 CSV 文件。
  3. 打印头部数据，进行人工 Quality Gate（质量门禁）校验，确保抓取到的视频（如 *Fast Fashion Environmental Impact*）确实契合我们的战略定义。
