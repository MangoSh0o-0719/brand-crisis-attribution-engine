# 🧠 Module 04: 工业级离线情感打标与量化推断引擎 (Quantized LLM Sentiment Inference Engine)

> **🎯 模块业务目标：**
> 本模块承接 03 模块的干净语料，依托 00 模块预先部署的离线 RoBERTa 大模型，对文本进行深度语义分析。为了突破 CPU 算力瓶颈，本项目启用了原生 PyTorch INT8 动态量化技术（Dynamic Quantization）。
> **🚨 核心指标准备**：模型将输出基础的 `sentiment_label` 与 `sentiment_score`。请注意，此处的 score 为【置信度 (Confidence)】，它将在下游 05 模块被严谨重构为【负面共识度 (Negative Consensus)】与【真实情感极性 (True Sentiment)】。



### 🔌 Cell 1 & 2 - 上游数据嗅探与环境挂载 (Upstream Handshake)
* **职责**：
  1. 挂载 `transformers`, `torch` 等深度学习框架库。
  2. 自动检索 03 模块输出的最新清洗数据 (`cleaned_comments`)，建立严格的输入输出版本控制（Timestamped Directories）。

### ⚡ Cell 3 - 核心引擎：离线模型加载与 INT8 动态量化 (INT8 Quantization Engine)
* **职责**：
  1. 强制断网，从本地 `models/roberta-sentiment` 目录加载分词器与权重。
  2. **🚀 算力降维打击**：调用 `torch.quantization.quantize_dynamic`，将庞大的 32 位浮点线性层（FP32）原地压缩为 8 位整数（INT8）。
  3. **工程收益**：在纯 CPU 环境下，实现模型体积缩小至 1/4，推断速度提升 2-3 倍，同时保持情感分类精度基本无损。
> *注：对于像 RoBERTa 这样拥有上亿参数的 Transformer 模型，学术界有一个共识：“大模型是严重过度参数化（Over-parameterized）的。” 这意味着模型内部有大量的冗余。即使你把权重的精度砍掉一截，它依然能保留 99% 的语义理解能力。

### 🏎️ Cell 4 - GPU/量化 CPU 加速的情感推断流水线 (Batch Inference Pipeline)
* **职责**：
  1. 摒弃低效的 `for loop` 逐条处理，启用大厂标准的 **微批处理架构 (Micro-Batching, BATCH_SIZE=32)** 喂入模型，最大化榨干 CPU/GPU 内存带宽。
  2. 实时监控推断进度（Tqdm），动态聚合三大阵营的 `POSITIVE`, `NEUTRAL`, `NEGATIVE` 标签与置信度。
  3. 输出带标签的结构化数据集 `sentiment_[group].csv`。

### 🛡️ Cell 5 - AI 输出质量门禁校验 (AI Output Quality Gate)
* **职责**：防止深度学习模型产生不可控的“黑盒灾难（幻觉或崩溃）”。
  1. **Schema 校验**：检查是否成功生成 `sentiment_label` 和 `sentiment_score` 字段。
  2. **完整性校验**：排查是否存在因文本过长（超出 512 Tokens）导致模型返回 Null 的崩溃数据。
  3. **标签污染校验**：确保分类结果严格锁定在三大基础类之中，拒绝任何格式脏数据流入下游 05 统计模块。
